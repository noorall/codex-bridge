# Agent Guidelines

## Commit Messages

- Follow the repository's existing Conventional Commit style: `type: short summary`.
- Use one of the established types when applicable: `feat`, `fix`, `ci`, `chore`, or `docs`.
- Use `fix` for bug fixes, `feat` for user-visible functionality, `ci` for workflow/build verification changes, `chore` for maintenance or release bookkeeping, and `docs` for documentation-only changes.
- Commit messages must be written in English.
- Commit messages must be a single-line English description.
- Commit messages must not contain line breaks, extra explanation, or redundant detail.
